Drive Snapshot - Was ist neu

V1.48 - Dezember 2019: Drive Snapshot 1.48

  • Anpassungen für aktuelle Betriebssysteme (Windows 10 (1909) und Server 2019)
  • Überarbeitung der Verschlüsselung

V1.47 - Juli 2019: Drive Snapshot 1.47

  • Anpassungen für aktuelle Betriebssysteme (Windows 10 (1809) und Server 2019)
  • Komplette Überarbeitung der Verschlüsselung. Images können jetzt mit einem vorher erzeugten Schlüssel verschlüsselt werden. Es ist nicht mehr notwendig ein Passwort im Klartext anzugeben. Die Verschlüsselung wurde auf AES256 umgestellt. Die Ver- und Entschlüsselung ist auf modernen Rechnern wesentlich schneller geworden.
  • Möglichkeit ein Medium eines Wechsesldatenträgers nach dem Abschluss des Backups auszuwerfen(--EjectMedium)
  • Bei einer Vollsicherung auf Netzwerklaufwerke wird die Hash-Datei zunächst lokal erzeugt und im Anschluss an das Backup in den Zielpfad kopiert. Das bsiherige Verhalten kann mit der Option --NoLocalHashFile wieder hergestellt werden.

V1.46 - März 2018: Drive Snapshot 1.46

  • Anpassungen für aktuelle Betriebssysteme (Windows 10 (1709))
  • Sehr viel schnelleres Testen von verschlüsselten Images
  • Zusätzliche Möglichkeit für ein schnelleres differentielles Image (--FastDiff:)
  • Möglichkeit ein USB-Laufwerk nach dem Abschluss des Backups zu entfernen (--EjectDrive)
  • Verbesserte Reparaturfunktion bei defekten Images (--repair)
  • Das Linux Partitionshandling wurde verbessert
  • Unterstützung neuerer Versionen des XFS (V5 mit CRC) Dateisystems

V1.36 - 20 Juli 2005: Drive Snapshot 1.36

     ältere Versionen von Drive Snapshot sind nicht kompatibel mit Windows 2003 SP1

Neu:  Backup:

  • Partitionen ohne Laufwerksbuchstaben (die über Mountpoints angesprochen werden)
  • Compaq/HP/Dell Service Partitionen
  • Linux Ext2/Ext3/Reiser/Swap Partitionen (diese können nicht gemountet werden)
  • Image Dateien werden so defragmentiert wie möglich abgespeichert
  • USB Sticks werden unterstützt
  • Ein bisschen Grafik im User Interface ;)

Restore:

  • auf alle obigen Partitionen
  • wenn die Rücksicherung unter Windows/BartPE erfolgt, kann die Dateisystemgröße der Partitionsgröße für NTFS Laufwerke angepasst werden.
    Vergrößern ist immer möglich; verkleinern nur bis zur Größe des höchsten belegten Sektors auf dem Laufwerk (nicht auf die belegte Größe der Dateien); im wesentlichen auf die Größe, die der Disk Defragmentierer anzeigt.
  • Im GUI können Partitionen erzeugt werden; versuchen Sie die rechte Mausttaste

beseitigte Fehler:

  • bisher waren nur Images, die in weniger als 256 Dateien aufgesplittet waren, mountbar.
    Jetzt werden bis zu 65535 Dateien (jede maximal 256 TB groß) unterstützt.
  • Unterstützung von Windows 2000/2003 Terminal Server
  • korrektes Verhalten, wenn Speicherkarten (MMC) nicht gesteckt sind
  • und natürlich diverser Kleinkram
